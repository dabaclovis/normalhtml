GIT

What is Git

Git is a distributed version control system that is used to track changes to files and coordinate work on those files among multiple people.

What is Version control

Version control is the management of changes to documents, programs, and other information stored as computer files. It is most commonly used in software development, where a team of people may change the same files.

Types of version control

There are two main types of version control systems: centralized and distributed. Centralized version control systems such as Subversion (SVN) use a single server to store all the versioned files, and team members can check out and check in files from the server. Distributed version control systems such as Git use a peer-to-peer network of repositories where each team member has a copy of the entire repository.

What is bash

Bash is a Unix shell and command language. It is the default shell in most Linux distributions, and is also available for macOS, Windows, and other operating systems. Bash is used to execute commands, create and modify files, and navigate the filesystem.

How to move files to staging area

To move files to the staging area, use the git add command. This command adds files to the staging area, which is a temporary storage area where files are tracked by Git before they are committed

Frequently used git Commands

  1. Git init: Initialize a local Git repository.
  2. Git status: Check the status of your current repository and see which files have been modified.
  3. Git add: Add files to the staging area.
  4. Git commit: Commit the staged files and add a message describing the changes.
  5. Git push: Push the committed changes to the specified remote repository.
  6. Git pull: Pull the latest changes from the specified remote repository.
  7. Git branch: List, create, or delete branches.
  8. Git checkout: Checkout a branch or files to work on.
  9. Git merge: Merge one or more branches into the current branch.
  10. Git remote: View and manage a list of remotes.
