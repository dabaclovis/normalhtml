MAVEN

What is Maven

Maven is a build automation tool used primarily for Java projects. Maven can also be used to build and manage projects written in C#, Ruby, Scala, and other languages. It is based on the concept of a project object model (POM) in which all the configuration for a project is stored. Maven provides developers with a complete build lifecycle framework to create a standardized way to build and release projects. It also provides a centralized repository for all project dependencies, making it easy to manage versions and dependencies

What is an Artifact

An artifact is a deployable component of a software system. It is a compiled, packaged, and versioned representation of a system's code, configuration, and/or data. Examples of artifacts include compiled source code, libraries, or executables.

What is a vulnerability

Vulnerability is a weakness in a system or its design that can be exploited by an adversary or attacker to gain unauthorized access to sensitive data, disrupt operations, or cause other damaging effects.

Maven global server

Maven Global Server is a hosted repository for software artifacts and packages. It provides storage for many popular open source software artifacts and can be used to provide access to artifacts from a single location.

Maven Local Server

Maven Local Repository is a local folder on your computer that is used to store all the project dependencies (library jars, plugin jars etc.). When maven build is executed, maven automatically downloads all the dependency jars into the local repository.

Maven Goals

There are server maven goals:

  • Clean
  • Package
  • Compile
  • Test
  • Install
