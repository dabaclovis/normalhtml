What is Ansible

Ansible is an open source automation platform used for IT tasks such as configuration management, application deployment, and orchestration. It can be used to manage both local and remote systems, and is designed to be very easy to use.

Ansible ad-hoc commands

Ansible Ad-hoc commands are one-time commands that can be used to perform quick tasks such as file transfers, package installation, and system administration.

E.g ansible all -m apt -a 'name=git, state=present' -b

The above command will install git on all managed nodes present in the inventory file:

  1. ansible: constant
  2. all: host/hosts machines present in the inventory file
  3. m: stands for the modules
  4. apt: is the modules use
  5. a: stands for arguement
  6. state: present means install
  7. state: absent means uninstall
  8. -b: equivalent to sudo in linux

Ansible Modules

Ansible Modules are reusable, standalone scripts that can be used by the Ansible API, or by the ansible command line tool. Modules can be written in any language and are used for executing specific tasks, like manipulating files, or installing packages.

Some modules of Ansible:

  • apt: To download packages from the internet
  • file: To make directory or file in a remote system
  • copy: To copy files from controller to managed nodes
  • git: To clone a remote repository
  • replace: To change remote host port of replace a file with some content
  • uri: To check the url of managed nodes
  • fetch: To copy files from managed nodes to controller machine
  • ping: To check if managed nodes are reachable
  • service: use to start a service in a managed node

Ansible Playbook

Ansible Playbooks are written in the YAML data serialization language and are used to define a set of tasks that will be executed on one or more remote hosts. Playbooks are written in a declarative manner, and can be used to orchestrate complex multi-tier deployments.
